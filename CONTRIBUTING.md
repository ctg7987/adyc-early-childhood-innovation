# Contributing Guide

Thank you for your interest in contributing to the ADYC Early Childhood Innovation project documentation!

## Who Can Contribute?

- **Original team members** – Please add your information and memories!
- **Hackathon participants** – Share photos, insights, or resources from the event
- **Domain experts** – Help expand the research sections with citations and data
- **Open source enthusiasts** – Improve documentation quality and structure

## How to Contribute

### For Team Members:

1. **Clone the repository:**
   ```bash
   git clone https://github.com/ctg7987/adyc-early-childhood-innovation.git
   cd adyc-early-childhood-innovation
   ```

2. **Create a branch:**
   ```bash
   git checkout -b add-[your-name]-contributions
   ```

3. **Make your changes:**
   - Update your profile in `team/roles.md`
   - Add any missing files or assets
   - Expand documentation with your insights

4. **Commit and push:**
   ```bash
   git add .
   git commit -m "Add [description of your changes]"
   git push origin add-[your-name]-contributions
   ```

5. **Submit a pull request** on GitHub

### For External Contributors:

1. **Fork this repository** on GitHub

2. **Make improvements:**
   - Fix typos or formatting
   - Add relevant research citations
   - Improve documentation clarity
   - Suggest structural enhancements

3. **Submit a pull request** with a clear description of changes

## Contribution Guidelines

### Documentation Standards:

- **Use clear, professional language**
- **Maintain existing structure** (don't remove placeholder sections)
- **Add citations** for research claims
- **Include image credits** for photos
- **Respect privacy** – Don't share sensitive information without permission

### File Naming Conventions:

- Use lowercase with hyphens: `customer-feedback.md`
- Be descriptive: `wireframe-parent-dashboard.png` not `img1.png`
- Include dates for time-sensitive docs: `interview-notes-2024-03.md`

### Markdown Formatting:

- Use headers appropriately (`#`, `##`, `###`)
- Include blank lines around headers and lists
- Use tables for structured data
- Add links to external resources
