# Screenshots Directory

This directory should contain screenshots of the working prototype demonstrated at the hackathon.

## What to Add

### Content Types:
- Screenshots of the working prototype
- Screen recordings (short GIFs)
- Interactive demo screenshots
- Mobile/desktop views
- Different user states/flows

### Suggested Organization:

```
screenshots/
├── mobile/
│   ├── home-screen.png
│   ├── activity-view.png
│   └── profile-page.png
├── desktop/
│   ├── dashboard-full.png
│   └── admin-panel.png
├── user-flows/
│   ├── onboarding-sequence.gif
│   └── complete-activity-flow.gif
└── feature-highlights/
    ├── gamification-example.png
    ├── progress-tracking.png
    └── ai-recommendations.png
```

## Screenshot Guidelines

### Best Practices:
- **High Resolution** – Capture at actual device resolution or higher
- **Clean UI** – Use realistic but anonymized data
- **Contextual** – Show features in action, not just static screens
- **Annotated** – Add brief captions or descriptions
- **Consistent** – Use the same device frames or styles

### Naming Convention:
```
[section]-[feature]-[state].png

Examples:
- parent-dashboard-empty-state.png
- child-activity-game-playing.png
- community-feed-scrolling.png
```

## Missing Assets?

If original screenshots are unavailable:
- Recreate from wireframes with annotations
- Use mockup tools to visualize the concept
- Create annotated diagrams showing intended functionality

## Instructions for Contributors

1. Place files in appropriate subdirectories
2. Add a `CAPTIONS.md` file explaining each screenshot
3. Consider creating a visual walkthrough document
4. Maintain privacy – blur or anonymize any sensitive data
